# Contributing

Contributions are welcome if they improve the educational value, security posture, or documentation quality of this project.

## Guidelines

1. Do not submit proprietary code, internal configurations, employer-specific designs, secrets, credentials, or confidential architecture details.
2. Use only public, open-source components.
3. Keep examples generic and educational.
4. Add tests or documentation for significant changes.
5. Follow secure coding practices.

## Suggested contribution areas

- MFA support
- Refresh token rotation
- Rate limiting
- Account lockout
- OpenTelemetry integration
- Threat model improvements
- Security test cases
